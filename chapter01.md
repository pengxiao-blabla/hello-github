# github分支（branch）

有一个网站可以可视化学习GitHub，直接在GitHub中搜索https://github.com/pcottle/learnGitBranching

**pull request** 简称PR，直接翻译为拉取请求，但是更好的翻译是**合并请求**:
贡献者把自己的分支请求合并进主干分支的提案。pull request 会比较两个分支之间的代码差异，然后仓库的管理员回来审核这个代码的改动（这个过程叫做code review，也就是代码审计）：
